- **Framework**: Next.js 15 with App Router and React Server Components
- **UI Library**: React 19 with Compiler optimizations
- **Styling**: Tailwind CSS v4 with custom design token system
- **Components**: shadcn/ui (Radix UI primitives)
- **State Management**: React Context + useReducer
- **Validation**: Zod schemas with TypeScript inference
- **Testing**: Vitest + React Testing Library

**Backend Stack:**
- **Framework**: FastAPI (async Python web framework)
- **Architecture**: Modular monolith with service layers
- **Validation**: Pydantic models
- **Session Store**: Redis (Docker container)
- **Testing**: pytest + pytest-asyncio

**Infrastructure:**
- **Containerization**: Docker + Docker Compose
- **Package Managers**: Bun (frontend), uv (backend)
- **Dev Experience**: Hot reload, auto-restart, volume mounts

## ğŸ’» Local Development (Without Docker)

### Frontend Setup

```bash
cd orchestratai_client

# Install dependencies
bun install

# Copy environment template
cp .env.example .env.local

# Edit .env.local with your values
# NEXT_PUBLIC_API_URL=http://localhost:8000

# Start dev server
bun dev

# Available at http://localhost:3000
```

### Backend Setup

```bash
cd orchestratai_api

# Install Python dependencies (Python 3.12 required)
pip install uv
uv sync

# Or use pip
pip install -r requirements.txt

# Copy environment template
cp .env.template .env

# Start dev server
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000

# API docs available at http://localhost:8000/docs
```

### Screenshot Capture Tips
**[Source: Epic 6 spec]**

**Desktop Screenshot:**
- Resolution: 1920x1080 or 1440x900
- Show all three panels populated with data
- Include at least 3-4 messages in chat
- Agent panel should show at least one ACTIVE agent
- Log panel should have 5+ entries visible
- Use realistic sample queries (not "test" or "hello")

**Mobile Screenshot:**
- Device: iPhone 12/13 size (375px width)
- Show Chat tab with conversation
- Take second screenshot showing Agents or Logs tab
- Ensure tab bar at bottom is visible
- Show footer stats bar if possible

**Tools for Screenshots:**
- macOS: Cmd+Shift+4 (select area)
- Windows: Snipping Tool or Win+Shift+S
- Browser: DevTools device emulation for mobile
- Optional: Use tools like [Screely](https://screely.com) for fancy mockups

**Image Optimization:**
- Format: PNG or WebP
- Size: Aim for <500KB per image
- Use ImageOptim (Mac) or TinyPNG (online) to compress
- Store in `docs/images/` directory
